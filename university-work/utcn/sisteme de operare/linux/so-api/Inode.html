Detailed Description

This class represents an interface to any of the hard disk's inodes. Thus it allows read/write access to the various inode elements. Here's the configuration of the inode which this class assumes:

Inode configuration
1 bit 7 bits 16 bytes 4 bytes 4 bytes 1 byte 14 bytes
inode valid/invalid file type checksum size of file
(in bytes)		 nr. of blocks
occupied by file		 size of file name reserved

Inode configuration (cont'd)
4 bytes 4 bytes 4 bytes 128 bytes 60 bytes rest up to include/defs.h::BLOCK_DIM
creation date last access date last update date file name 15 direct block addresses nr. of left bytes/4 indirect block addresses

void  readFromHardDisk (HDD* hdd, unsigned long blockNo)
throw( HardDiskNotInitializedException*, IOException*, InvalidBlockNumberException*, InvalidDateException*)

readFromHardDisk

This method reads the inode from block number blockNo of the hdd hard disk and initializes all of the class' data members according to the inode configuration scheme presented above. Beware that if isValid() returns false after a read it means that all the rest of the data is just garbage. Do not use it without previously checking whether the inode is valid or not.

If valid is true and either one of creation date, last access date or last update date is an invalid date an InvalidDateException is thrown, but all the data members are properly initialized. The invalid date(s) will be initialized with the current date. If such a behavior is not what you want, you can take whichever action you desire in the exception handling code.

If the size of the file name on disk is greater than 128, the internal data member will be set to 128.

The method throws:

  • HardDiskNotInitializedException* forwarded from hdd.
  • IOException* forwarded from hdd.
  • InvalidBlockNumberException* forwarded from hdd.
  • InvalidDateException* if valid is true and either one of creation date, last access date or last update date is an invalid date.

Parameters:
hddthe hard disk where to read the inode from.
blockNothe number of the hard disk block to read.

void  writeToHardDisk (HDD* hdd, unsigned long blockNo)
throw(HardDiskNotInitializedException*, IOException*, InvalidBlockNumberException*)

writeToHardDisk

This method writes the inode as the blockNoth block of the hdd hard disk with its data member values according to the inode configuration scheme presented above.

If any of the array data members is NULL, zero (0) characters will be written for the entire length of the array. If any of the dates is NULL, the current date will be used.

The method throws:

  • HardDiskNotInitializedException* forwarded from hdd.
  • IOException* forwarded from hdd.
  • InvalidBlockNumberException* forwarded from hdd.

Parameters:
hddthe hard disk to write the inode to.
blockNothe number of the hard disk block to write.

unsigned char*  getAsBytes ()

getAsBytes

This method returns the inode in the exact same manner as it is stored on the hard disk (as an array of include/defs.h::BLOCK_DIM unsigned chars).

If any of the array data members is NULL, zero (0) characters will be written for the entire length of the array. If any of the dates is NULL, the current date will be used.

Returns: the inode as it is stored on the hard disk.

serialize

This method returns a serialized representation of the inode (as it's supposed to appear on the hard disk).

If any of the array data members is NULL, zero (0) characters will be written for the entire length of the array. If any of the dates is NULL, the current date will be used.
