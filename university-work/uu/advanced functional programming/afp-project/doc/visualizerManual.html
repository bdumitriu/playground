General guidelines

Compiling

The visualizer must be compiled before it can be used. We recommend using GHC version 6.2.2 or higher, and the command to be invoked (at the top level directory) is:
ghc -icommon -package wx -O1 --make -o visualizer.exe Visualizer
You can replace visualizer.exe with whatever output filename you want. The optimizations -O1 are not required but are recommended.

Invoking

The usage for the visualizer is:
Usage: <program-name> -w worldFile -r redAntFile [-b blackAntFile] [-c numberOfRounds][-s randomSeed] [-o]
  -w FILE --world=FILE the world file to use
  -r FILE --red-ant=FILE the ant code file for red ants
  -b FILE --black-ant=FILE the ant code file for black ants (optional)
  -c INT --nr-rounds=INT the number of rounds to simulate (optional)
  -s INT --random-seed=INT the initial seed for the RNG (optional)
  -o --output use if you want to dump output (optional)
 If no code is supplied for the black ants, then they will do nothing all the time.

Using the visualizer

Board description

The image below shows an initial screenshot from the visualizer:

The basic layout consists of a menu panel on the left and the board drawn on the right with a small text line below it displaying information related to each cell. Placing the mouse over a cell will show its relevant information on this text line.
The board itself is quite explicit, and the symbols mean:

  • White hexagon -> Clear cell
  • Brown hexagon -> Rocky cell
  • Green hexagon -> Cell with food
  • Black triangle -> Black ant
  • Red triangle -> Red ant
  • A small white dot over an ant -> The ant is carrying food
  • Pink hexagon -> Red anthill
  • Dark grey hexagon -> Black anthill

Panel description

The markers set by the ants are not shown in the board, but in the panel:

  • Step -> Steps one turn
  • Start -> Starts cycling turns until the user presses stop
  • Stop -> Stops turn cycling
  • Refresh -> Redraws the board. Useful if that is not automatically done while cycling turns
  • Jump to turn -> Jump to the turn number specified in the input below this button
  • + -> Increase cycling speed
  • - -> Decrease cycling speed
Below the "Jump to turn" button, an input shows the current turn number. If you change it and press the "Jump to turn" button, then all buttons will become inactive while the program shifts to that turn.
While moving the mouse over a cell, the markers are also displayed here, in the checkboxes. A checked box means that a marker is set, namely by the red team (RM) or black team (BM).
